## The Story

*"You're a SaaS vendor. Your AI assistant serves both Acme Corp and Globex Inc. When an Acme employee asks the agent a question, FGA ensures it only queries Acme's data — even though Globex's data sits in the same database. No multi-tenant bugs in the agent code; Okta FGA is the enforcement layer."*

P7 demonstrates **fine-grained authorization for multi-tenant SaaS AI**. A single agent instance serves multiple enterprise customers. Okta FGA enforces tenant isolation at the tool-call level — the agent never needs to implement tenant-filtering logic itself.

---

## Intended Architecture

```
Tenant user (Acme)
  ↓ PKCE login
  id_token { sub: alice@acme.com, org_id: acme }
  ↓
p7-saas-agent
  ↓ extract org_id from token
  ↓ before each tool call:
      POST /stores/{FGA_STORE_ID}/check
        user:   alice@acme.com
        object: tenant:acme
        relation: member
  ↓ Okta FGA returns allow / deny
  ↓ allow → tool call with org_id=acme filter
  ↓ deny  → "Access denied for this resource"
```

---

## Intended Flow

```
1.  Tenant user authenticates via PKCE
    → id_token carries a tenant claim (e.g. org_id=acme)
2.  Console stores id_token in HttpOnly cookie; sends as Bearer to p7-agent
3.  Agent extracts tenant ID from the id_token payload
4.  Before every tool call, agent checks Okta FGA:
      "Does user <sub> have access to resource <tool> for tenant <org_id>?"
5.  FGA returns allow/deny based on the relationship graph:
      user alice@acme.com → tenant acme → resource hr_data
6a. Allow → agent calls tool with tenant-scoped query (org_id=acme filter injected)
6b. Deny  → agent returns: "You don't have access to this resource for your organization."
7.  No data from tenant B is ever returned to tenant A — enforced at FGA layer
```

---

## FGA Relationship Model (Planned)

```
type user
type tenant
  relations
    define member: [user]
type resource
  relations
    define can_read: [user] or member from tenant
```

Sample tuples:
```
(user:alice@acme.com, member, tenant:acme)
(user:bob@globex.com, member, tenant:globex)
(tenant:acme, can_read, resource:hr_data)
(tenant:globex, can_read, resource:hr_data)
```

Alice can read Acme's HR data. Bob can read Globex's HR data. Neither can read the other's.
